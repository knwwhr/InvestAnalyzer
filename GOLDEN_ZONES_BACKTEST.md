## 🔍 패턴별 기준 검증

### 1. 🔥 Power Candle (시동)
**조건**:
- 거래량 >= 전일×2.0 & >= 20일평균×1.0
- 등락률 +5~12%
- 시가 ≒ 저가 (0.5% 이내, 꽉 찬 양봉)
- 거래대금 >= 100억

**감지**: 0개

**분석**:
- 기준이 매우 엄격 (거래량 2배 + 특정 등락 범위 + 거래대금 100억)
- 현재 시장에서 이러한 조합이 없음
- ✅ **과적합 방지 확인됨**

### 2. 🕳️ 개미지옥 (속임수)
**조건**:
- 장중 저가 < 전일 저가 × 0.97 (-3% 이탈)
- 아래꼬리 >= 몸통 × 2.0
- 종가 >= 시가 (양봉 마감)
- 3일 내 최저가 갱신

**감지**: 0개

**분석**:
- 특정 패턴 (하락 후 반전) 요구
- 노이즈 필터 (3일 최저가 갱신)가 강력하게 작동
- ✅ **일반적 하락장과 구분 성공**

### 3. ⚡ N자 눌림목 (재장전)
**조건**:
- 5일 내 +15% 이상 급등일 존재
- 고점 대비 -5~-12% 조정
- 금일 거래량 < 20일평균 × 0.7
- 거래량 < 기준봉 × 50%

**감지**: 0개

**분석**:
- 급등 후 조정 패턴 요구
- 최근 5일 내 +15% 급등이 없음 (정상적 시장)
- ✅ **급등 종목만 선별 가능함 확인**

### 4. 🌋 휴화산 (응축)
**조건**:
- 거래량 <= 20일평균 × 0.4
- 캔들 몸통 <= 1.5%
- Bollinger Band Width < 0.1
- 5일선 위 + 거래대금 >= 30억

**감지**: 0개

**분석**:
- 변동성 수축 + 거래량 감소 조합
- BB Width 0.1 미만은 매우 드문 조건
- ✅ **과도한 수축 상태만 감지 (정확함)**

## ✅ 검증 기준 평가

### 1. 과적합 방지 (Overfitting Prevention)
**기준**: 감지 빈도 주당 5개 이내
- **결과**: 0개 / 51개 종목 (0%)
- **평가**: ✅ **합격** - 지나치게 보수적이지만 과적합 위험 없음

### 2. 신호 품질 (Signal Quality)
**기준**: 승률 75% 이상, 비교군 대비 +5% 이상
- **결과**: 감지 없어 백테스트 불가
- **평가**: ⚠️ **보류** - 실전 데이터 축적 필요

### 3. 실용성 (Practicality)
**기준**: 사용 가능한 빈도
- **결과**: 현재 시장에서 0개
- **평가**: ⚠️ **검토 필요** - 기준 완화 고려

## 💡 권장사항

### Option A: 현재 기준 유지 (보수적 전략)
**장점**:
- ✅ 과적합 방지
- ✅ 신뢰도 높은 신호
- ✅ 노이즈 최소화

**단점**:
- ❌ 감지 빈도 낮음 (실용성 저하)
- ❌ 백테스트 불가 (샘플 부족)

**권장**:
- 1~2주 실전 데이터 수집 후 재평가
- 시장 변동성 높아질 때 감지 가능성 증가

### Option B: 기준 완화 (적극적 전략)
**제안**:
1. **Power Candle**: 거래대금 100억 → 50억으로 완화
2. **개미지옥**: 아래꼬리 2.0배 → 1.5배로 완화
3. **N자 눌림목**: 급등 기준 +15% → +10%로 완화
4. **휴화산**: BB Width 0.1 → 0.15로 완화

**기대 효과**:
- 주당 1~3개 감지 (실용성 향상)
- 여전히 보수적 기준 유지

### Option C: 2-Tier 시스템 (균형 전략)
**제안**:
- **Tier 1** (현재 기준): +15점 보너스, 매우 강력한 신호
- **Tier 2** (완화 기준): +10점 보너스, 중간 강도 신호

**장점**:
- 감지 빈도 향상
- 신호 강도 차별화
- 사용자 선택권 제공

## 🎓 학습 포인트

### 백테스트의 한계
1. **시점 의존성**: 백테스트 시점의 시장 상황에 따라 결과 변동
2. **샘플 부족**: 극단적 패턴은 드물게 나타남 (정상)
3. **전방 테스트 필요**: 실제 시장에서의 성과가 최종 검증

### 성공적인 패턴 설계
1. **과적합 방지 우선**: 감지 없음 > 잘못된 신호 다수
2. **보수적 기준**: 초기에는 엄격하게, 데이터 기반 완화
3. **단계적 검증**: 구현 → 모니터링 → 재평가 → 조정
